# Daily Changelog - 2026-04-06 (Monday)

## Summary
Implemented deployment automation system with SSH-based remote deployment, cross-platform build support, and service management without sudo requirements.

## Changes

### Added
- Deployment script (`deploy/deploy.sh`) with SSH-based remote deployment
- Automatic OS and architecture detection for cross-platform builds
- Backup system with configurable retention (`DEPLOY_KEEP_BACKUPS`)
- `deploy/.env.example` with deployment configuration template
- `deploy/README.md` with deployment documentation
- `host` field in server config for custom bind addresses
- `API_BASE_URL` environment variable for frontend-backend communication
- Database connection test (`backend/database/database_test.go`)

### Changed
- Server now reads `host` from config.yaml (defaults to 0.0.0.0)
- Makefile `build` target copies and configures config.yaml for production
- Makefile supports `API_BASE_URL` parameter for frontend builds
- Service management uses `nohup` instead of systemd (no sudo required)
- Frontend vite config supports `API_BASE_URL` environment variable

### Fixed
- Deployment script tar path issues (using basename)
- Service installation failures on servers without sudo access

### Removed
- Systemd service file installation (replaced with nohup)

## Impact
- Enables one-command deployment: `make deploy`
- Supports deployment to any Linux/macOS server without sudo requirements
- Automatic backups protect against deployment failures
- Cross-platform builds work on amd64 and arm64 architectures
